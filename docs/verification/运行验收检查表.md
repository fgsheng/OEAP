# 运行验收检查表（SRE视角）

> 目的：在上线/变更后验证“可用、可观测、可回滚”。
> 范围：日志、指标、追踪、告警（含SLO/错误预算）。

## 1. 基线与范围确认
- [ ] 变更范围、影响面、回滚方案已确认
- [ ] 关键用户路径与依赖系统清单已更新
- [ ] SLO/SLI 定义与阈值已同步（可用性/延迟/错误率/饱和度）

## 2. 日志（Logs）
- [ ] 结构化日志（trace_id/request_id/user_id）
- [ ] 时间戳统一且含时区
- [ ] 分级合理（INFO/WARN/ERROR）
- [ ] 关键错误有码或可聚合字段
- [ ] 采集链路完整

## 3. 指标（Metrics）
- [ ] 延迟 p50/p95/p99
- [ ] 流量 QPS/并发
- [ ] 错误率 5xx/超时/业务错误
- [ ] 饱和度 CPU/内存/队列
- [ ] SLO 面板可用，错误预算告警配置

## 4. 追踪（Tracing）
- [ ] 入口请求覆盖率达标
- [ ] 跨服务 trace_id 贯通
- [ ] DB/缓存/外部API span

## 5. 告警（Alerting）
- [ ] 负责人/值班路由
- [ ] 阈值与抑制策略
- [ ] Runbook 链接
- [ ] 5xx/延迟/可用性告警模拟验证

## 6. 运行验证
- [ ] 关键路径端到端成功
- [ ] 依赖失败时可用性符合预期
- [ ] 回滚演练

## 7. 产出与签署
- [ ] 验收记录归档
- [ ] 同步结果
- [ ] 问题建单

最低门槛示例：SLO≥99.9%，p99≤300ms，错误率≤0.1%。
