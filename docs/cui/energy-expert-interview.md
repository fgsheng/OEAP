# Energy Expert Interview — CUI Script (Sales Engineer)

## 目标
- 以技术预售（Sales Engineer）视角完成专家访谈，验证能源行业场景中的关键痛点、数据来源、系统集成与成功衡量指标。
- 为后续产品演示/POC提炼“可证明的价值点”。

## 角色
- **SE（Sales Engineer）**：技术预售，负责需求澄清与技术可行性验证
- **EE（Energy Expert）**：能源行业专家/运营与规划负责人

---

## CUI 对话脚本

**SE**：感谢你参加。为了尊重时间，我想把这次访谈聚焦在三个点：你们的核心业务目标、当前系统和数据流、以及衡量成功的指标。可以吗？

**EE**：可以。

**SE**：先从业务目标开始。你们当前最想解决的能源相关问题是什么？是成本、可靠性、还是合规？

**EE**：主要是成本与预测准确性。我们需要更准确的负荷预测，才能优化采购与调度。

**SE**：理解。我们在做此类项目时，最关键的价值点通常是“减少采购溢价”和“降低偏差成本”。你们今天预测偏差大概在什么范围？

**EE**：月度偏差大约在 8%–12%。

**SE**：这已经是可量化的目标了。我们后面可以把 POC 成功标准设成“将偏差控制在 5%以内”。接下来我想了解数据与系统。你们的负荷数据来自哪些系统？

**EE**：SCADA、能源管理系统（EMS）、以及财务的采购系统。数据频率不一样，格式也不一致。

**SE**：这通常会引发集成复杂度。是否有现成的数据中台或数据湖？还是各系统各自为政？

**EE**：有数据湖，但标准化程度不高。

**SE**：明白。那如果我们要落地，关键在于两点：
1) 是否能在你们的数据湖中读取并做统一建模；
2) 是否能与 EMS 的调度接口对接，实现建议或自动化。

**EE**：对，尤其是第二点。

**SE**：关于对接，EMS 有没有现成的 API 或消息总线？

**EE**：有 API，但权限管理严格，需要走安全评审。

**SE**：安全评审是标准流程。我们会准备包含加密、审计、访问控制的技术说明，并提供最小权限的接口方案。

**EE**：可以。

**SE**：还有一个点是实时性。你们需要实时优化，还是日内/日前即可？

**EE**：主要是日内预测和调度优化。

**SE**：很好。这样我们可以将 POC 范围限定在“日内预测 + 调度建议”，避免范围扩散。

**EE**：赞同。

**SE**：最后是成功衡量指标。除了预测偏差下降，你们还会关注哪些指标？比如采购成本、负荷响应时间或合规指标。

**EE**：采购成本与调度响应速度。

**SE**：了解。我们会把 POC 成功指标定义为：
- 预测偏差 ≤ 5%
- 采购成本下降（以历史基线对比）
- 调度建议输出时间 ≤ X 分钟

**EE**：这个可以。

**SE**：很好。接下来我会整理一份 POC 方案草案和数据清单，确认范围与成功标准。我们在技术层面会先验证数据对接与模型可行性，再进入调度建议的闭环。

**EE**：期待。
