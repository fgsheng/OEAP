# 平台总体架构与集成模式（Software Architect）

## 0) 目标与约束
- 目标：可扩展、易集成、高可用平台架构
- 约束：低耦合、可演进、可观测；同步与异步混用

---

## 1) 组件交互
**核心组件**
- API Gateway/Edge：鉴权、限流、路由
- 服务层（业务域服务）
- 事件总线
- 数据层（事务库 + 查询库）
- 集成层（Integration Hub）
- 可观测性组件

**交互方式**
- 同步：Gateway → 业务服务（HTTP/gRPC）
- 异步：业务服务 → 事件总线 → 订阅者
- 内部编排：集成层负责流程编排

---

## 2) 事件与同步机制
**事件类型**
- 领域事件：业务状态变化
- 集成事件：外部系统同步触发

**机制设计**
- Outbox + CDC：事务一致性
- 事件重放：补偿与灾备
- 幂等性：消费者幂等处理
- 同步策略：强一致/最终一致分离

---

## 3) 可扩展性与高可用
**可扩展性**
- 无状态化，水平扩展
- 读写分离 + 缓存
- 分区/分片（租户/业务维度）

**高可用**
- 多副本 + 负载均衡
- 多可用区部署
- 消息中间件持久化与重试
- 灾备：主备或多活

**故障隔离**
- Circuit Breaker + Bulkhead
- 超时与退避重试

---

## 4) 外部系统集成（CLI → MCP）
**流程**
1. 外部系统提供 CLI
2. MCP Server 适配：认证、审计、限流
3. 平台集成层调用 MCP
4. 结果回传并触发内部事件

**优势**
- 低侵入、统一协议、易扩展

**风险与对策**
- CLI 不稳定 → 重试与健康检查
- 命令注入 → 参数白名单 + 沙箱
