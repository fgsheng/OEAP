# 专家输入→知识/技能/工具沉淀流程（Workflow Architect）

## 1) 引导式采集步骤
**STEP 1：输入场景界定**
- 采集项：业务目标、适用场景、输入/输出边界、频率、触发条件
- 恢复动作：回问目的/边界/触发/受众

**STEP 2：关键流程拆解**
- 采集项：步骤序列、依赖条件、决策点、失败分支、回退动作
- 恢复动作：追问“如果…会怎样？”、“前置条件是什么？”

**STEP 3：知识粒度归类**
- 采集项：规则/策略/范式/操作步骤/工具调用/模板/示例
- 恢复动作：按类型分段重述

**STEP 4：证据与例子**
- 采集项：样例输入/输出、反例、异常处理
- 恢复动作：要求最小可复现例

**STEP 5：约束与假设**
- 采集项：依赖系统/工具、权限、环境、频率、SLA
- 恢复动作：列出所有假设并标记风险

---

## 2) 缺失信息检测
**检测维度（缺失即回问）**
- 触发条件
- 输入/输出
- 决策点
- 失败与恢复
- 工具/权限
- 样例

产出：缺失信息清单（Gap List）+ 影响级别

---

## 3) 确认回路
- 复述确认
- 风险确认
- 变更确认（变更摘要 + 专家批准）

---

## 4) 结构化表示
**必填字段**
- 标题 / 版本 / 状态 / 适用范围
- 触发条件
- 输入/输出（Schema）
- 过程步骤（含分支）
- 失败模式与恢复
- 依赖与权限
- 示例（正/反）
- 假设与风险
- 适用/不适用边界

**格式**
- Markdown + YAML front matter
- JSON Schema / OpenAPI / MCP tool schema
- Mermaid 流程图 + 决策树

---

## 5) 质量门控
- Gate1：完整性
- Gate2：可执行性
- Gate3：可验证性
- Gate4：可维护性

---

## 6) 版本化策略
- MAJOR：逻辑与输入输出变化
- MINOR：新增分支/步骤
- PATCH：文案修订/示例补充

生命周期：Draft → Review → Approved → Deprecated

---

## 7) 与本体/Skill/MCP 映射
| 资产类型 | 表达形态 | 目标载体 | 触发方式 | 产出形式 |
|---|---|---|---|---|
| 知识规则 | 规则/判断 | 本体 | 推理/问答 | 事实/结论 |
| 操作流程 | Step+分支 | Skill | 任务请求 | 可执行流程 |
| 工具调用 | API/CLI | MCP Tool | 自动/显式 | 结构化响应 |

映射规则：
- 可重复执行的动作 → Skill
- 需要外部系统访问 → MCP Tool
- 概念/规则/关系 → 本体
