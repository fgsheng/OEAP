# 产品包结构与版本策略（草案）

## 1) 产品包结构（建议）
```
package.yaml
ontology/
  schema.json
  entities.json
skills/
  skill-manifest.yaml
  skill-*.md
mcp/
  mcp-manifest.yaml
  tools/
    tool-*.yaml
artifacts/
  examples/
  tests/
```

## 2) 元数据（package.yaml）
- name
- version (SemVer)
- owner
- domain
- tags
- created_at
- updated_at
- dependencies

## 3) 版本策略
- MAJOR：输入/输出或流程逻辑变化
- MINOR：新增技能/工具/流程
- PATCH：文案修订/样例补充

## 4) Git 存储
- 每个产品包一个仓库或 monorepo 子目录
- tag 版本号
- changelog 记录
