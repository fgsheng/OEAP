# 服务拆分与数据模型方案（Backend Architect）

## 1) 核心服务边界
1. **IAM**：用户/组织/角色/权限/Token
2. **内容/知识服务**：文档、版本、索引
3. **图谱服务**：实体关系、推理路径
4. **检索与向量服务**：向量化、混合检索
5. **协作与工作流服务**：任务、审批、评论
6. **审计与合规服务**：全链路日志
7. **API Gateway**：鉴权、限流、路由、版本
8. **通知服务**：站内信/邮件/Webhook

## 3) 数据模型概要
### PostgreSQL
- users(id, email, pwd_hash, org_id, status, created_at)
- orgs(id, name, plan, created_at)
- docs(id, org_id, owner_id, title, status, created_at)
- doc_versions(id, doc_id, version_no, checksum, created_at)
- doc_chunks(id, doc_id, version_id, content, idx, created_at)
- tasks(id, org_id, assignee_id, status, due_at)
- comments(id, task_id, author_id, content, created_at)
- audit_events(id, trace_id, actor_id, action, target, ts)

### Neo4j
- Entity(id, type, name, meta)
- REL(type, from, to, weight, created_at)

### Qdrant
- collection: doc_chunks
  - payload: {doc_id, version_id, chunk_id, tags, lang}
  - vector: embedding
  - index: HNSW, cosine

### Redis
- sessions:{token} -> user_id
- rate_limit:{user}:{route}
- doc_cache:{doc_id} -> latest
- feature_flags

## 4) 关键实体与关系
- User ↔ Org
- Doc → Versions → Chunks
- Doc/Chunk ↔ Entity
- Entity ↔ Entity
- User → Task/Comment
